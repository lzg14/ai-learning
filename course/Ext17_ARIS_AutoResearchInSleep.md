# Ext17：ARIS——让 Claude Code 在你睡觉时做科研

---

## 一、背景：Agent 科研的可信性危机

自主科研 AI Agent 已经能够跑通"从灵感到论文"的完整流程——一觉醒来，Agent 也许已经跑完实验、甚至写出一篇有模有样的论文了。

但问题是：**我们怎么知道 Agent 有没有在结论里悄悄"说谎"？**

当前 Agent 仍存在两个关键问题：

1. **同模型系列生成与审查**：许多系统性错误因此难以在内部暴露
2. **长程无监督推理**：Agent 在几乎无人监督的情况下连续工作数天后，其最终结论是否真的有充分证据支撑，难以判断

上海交通大学团队提出了 **Auto-Research-in-Sleep（ARIS）**，一个用于自动化科研的开源研究 Harness。论文链接：https://arxiv.org/abs/2605.03042

> **核心理念**：ARIS 的重点不是让 Agent 更快地写出论文，而是让写出的论文更经得起检验。

---

## 二、系统拓扑：三层架构

ARIS 的系统由三层构成：

| 层次 | 作用 | 构成 |
|------|------|------|
| **执行层** | 提供具体能力 | 可复用的 Markdown 定义技能 + 持久化研究 Wiki |
| **编排层** | 串成完整流程 | 5 个端到端工作流，覆盖发现→实验→成稿→投稿后 4 阶段 |
| **保障层** | 证据审计与文稿检查 | 3 阶段证据-声明审计级联、5 遍科学写作编辑流水线、数学证明检查器、视觉 PDF 审查、引用审计 |

> 保障层是 ARIS 最核心的创新。

---

## 三、核心机制：跨模型对抗协作

研究团队判断：单个 Agent 难以可靠地完成长程研究任务。因此采用跨模型家族的"执行-审查-修正"循环：

| 角色 | 默认模型 | 职责 |
|------|----------|------|
| **执行者** | Claude 家族 | 产出代码、实验或论文草稿 |
| **审查者** | GPT-5.4 家族 | 在预定义评分标准下打分，返回结构化行动项 |

循环流程：执行者产出 → 审查者打分 + 结构化行动项 → 执行者修正 → 再次提交 → 直到评分达标。

---

## 四、端到端工作流

5 条工作流覆盖从发现到投稿后的完整研究阶段：

| 工作流 | 覆盖阶段 | 核心内容 |
|--------|----------|----------|
| **W1 创意发现** | 发现 | 文献调研、新颖性检查、实验规划（带审稿人门控） |
| **W2 实验桥接** | 实验 | 代码实现、算力执行、结果回收（带代码审查和自动调试回退） |
| **W3 自动审查循环** | 成稿 | 跨模型审查评分、提取行动项、按需 GPU 实验、修订章节、检查收敛 |
| **W4 论文写作** | 成稿 | 7 步骤：论文规划→图表生成→LaTeX 撰写→5 轮编辑→证明检查→结论审计→编译；再经 2 轮 GPT-5.4 xhigh 视觉审查进入自动改进循环 |
| **W5 投稿后** | 投稿后 | 解析审稿意见→拆分关键问题→规划回应策略→起草回复→三道安全检查（防捏造/过度承诺/遗漏回应）→压力测试→定稿 |

---

## 五、保障层：证据-主张审计链

ARIS 最具特点的设计——3 步审计链，确保论文结论有充分证据支撑：

### 5.1 三阶段审计级联

| 阶段 | 名称 | 检查内容 |
|------|------|----------|
| **Stage 1** | 实验审计 | 检查伪标签、幽灵结果、未执行指标、范围外推等可靠性失效模式 |
| **Stage 2** | 结果→主张 | 将每条候选结论与现有证据逐一对应，判定为"有支持""部分支持"或"不成立"；审计失败的主张直接降级 |
| **Stage 3** | 论文主张审计 | 全新审查者（不带上下文）将稿件每条定量主张与主张台账及原始结果文件逐一比对 |

### 5.2 五轮科学编辑

初稿完成后，ARIS 会进行 5 轮依次编辑：冗余表达 → 主动语态 → 局部连贯 → 术语一致性 → 数字一致性。

### 5.3 额外保障机制

- **数学证明检查器**：对理论成分较重的论文审查证明义务
- **视觉 PDF 审查**：检查图注错位、版面异常、表格可读性
- **引用审计**：检查文献是否存在、元数据是否正确、引用是否真的支持正文论断

### 5.4 W5 三道安全检查

投稿后阶段防三类问题：捏造、过度承诺、遗漏回应。

## 六、效果数据

- 技能库已从 21 个核心技能扩展到 **65 个以上**，覆盖机器人学、硬件设计、通信、数学证明、基金申请、演示生成等方向
- 已在 Claude Code、Codex CLI 和 Cursor 三个平台完成测试
- 审查端可接入 GPT、Gemini、DeepSeek 等多种模型后端
- **真实隔夜运行记录（约 8 小时）**：4 轮"审查-修改"循环，内部审查分数从 5.0/10 提升到 7.5/10，触发 20+ 次 GPU 实验，主动删除缺乏证据支撑的结论

> 研究团队特别强调：以上均为观察性证据，不能做因果判断——只能说明"审查驱动修改"可在真实流程中操作化，不能证明跨家族审查一定优于同家族审查。

## 七、总结与反思

### ARIS 的本质

ARIS 真正推进的不是自动化科研的速度，而是它的**可信性**。它未必已经给标准答案，但至少把一个长期被忽视的问题摆到了台面上——对自动化科研来说，这或许比再快一点、更自动一点更重要。

### 核心不足

| 局限 | 说明 |
|------|------|
| **缺乏受控评估** | 所有结果均为观察性记录，无法在因果层面归因于 ARIS 本身 |
| **不保证输出正确** | 三阶段审计链可拦住常见失真，但不能保证发现所有错误或伪造 |
| **迎合审查风险** | 若审查者持续偏好某种方法论，系统可能优化对审查口味的迎合而非真正科学质量 |
| **人类仍需负责** | 最终研究方向选择、证据核验和提交决定仍需由人负责 |

### 未来方向

- **Harness 自我进化**：Agent 自动分析历史失败案例，提出规范改进建议
- **跨项目模板化**：将 5-Stage Pipeline、Review Loop 抽象为可参数化模板
- **更精细的 Agent 角色矩阵**：Performance Auditor、Security Scanner、Documentation Sync Agent

### 更大价值

ARIS 这套机制未必只服务于论文写作——审查者独立、"证据到结论"审计和可追溯台账，理论上可置于模型输出与后续训练信号之间，成为**自我改进系统的一层显式监督**。
