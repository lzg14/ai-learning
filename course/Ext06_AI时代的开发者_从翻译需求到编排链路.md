# Ext06：AI时代的开发者——从翻译需求到编排链路

> 当软件载体消失，代码变成一次性的，开发者的价值在哪里？本文从「软件是什么」这一根本问题出发，探讨 AI 时代开发者角色的重构。

---

## 一、软件是什么：剥掉外壳之后剩什么？

不同时期人们说的「软件」，其实是不同的东西：

- PC 时代：软件是窗口
- 移动互联网时代：软件是 App
- SaaS 时代：软件是订阅制的网页

大家把自己熟悉的软件外壳，当成了软件本身。

尝试把外壳剥掉——软件剩的是什么？

**做事的能力。**

- 人们安装 Word，用的是文字处理能力
- 用 Photoshop，用的是图像编辑能力
- 订阅飞书，订阅的是办公管理能力

菜单、按钮、页面只是能力暴露给人的操作面板。点按钮、填字段、切页面，只是走向目标的路径。

### 软件在干什么？

把原本长在人身上的**过程性知识**剥离出来，封装起来，复制出去：

- 会计知道怎么记账 → 后来变成财务软件
- 销售知道怎么跟客户 → 后来变成 CRM
- 运营知道怎么发活动 → 后来变成营销自动化

### SaaS 的本质

CRM 出售销售流程，ERP 出售资源管理流程，飞书和 Slack 出售协作流程。企业买 SaaS，买到的是一套「公司应该如何运转」的**默认答案**。

一套报销系统规定谁能报、报多少、怎么审、谁负责。一套 CRM 规定客户怎么分配、商机怎么推进、销售怎么汇报。**制度写在纸上经常被忽略，写进软件就会自动执行。**

> SaaS 软件，更像是把制度打包，然后出租。

## 二、软件的进化位置

最早的计算机没有软件。ENIAC 时代，每算一个新任务，工程师得画好几天重新接线——硬件只认电路，人脑里的意图没法直接灌进去。

1945 年冯·诺依曼提出「存储程序」方案：把指令和数据一起存进内存，机器按顺序读指令执行。这层可以随时替换的逻辑，就是软件的起点。

**从那以后，所有软件做的事情都没变过：站在人和机器中间，把人想做的事翻译成机器能执行的步骤。**

每一次形态变化，都是在降低「翻译」的门槛：

| 时代 | 连接什么 |
|------|---------|
| 早期软件 | 人 → 计算机 |
| 互联网软件 | 人 → 信息 |
| SaaS | 人 → 组织流程 |
| Agent 软件 | 人、模型、工具、数据、权限、组织、外部世界 |

> 软件正在从单点工具变成跨系统的协调器。

### 价值链位置决定你的价值

- **站在意图入口**：控制需求
- **站在执行层**：控制履约
- **站在数据层**：控制上下文
- **站在权限层**：控制边界
- **站在 Skill 层**：控制方法

> Agent 时代的竞争，会从抢页面入口升级成抢任务链路里的关键位置。

## 三、AI 带来的三个跨越

### 跨越一：从明确操作到模糊意图

之前的软件只能响应明确操作——用户必须知道点哪里、填什么、选哪个选项。

AI 软件开始接收模糊意图：用户可以说「帮我做成老板能看懂的版本」。从这一刻起，**软件开始承担理解任务的责任**。

### 跨越二：从固定输出到开放结果

- 传统软件输出预设格式：表格、报表、订单、审批状态、PDF
- AI 软件输出开放：文章、图片、网页、代码、方案、临时工具

输出形态由任务决定，不再由系统预设。

### 跨越三：从单步功能到多步任务

- 传统软件执行**功能**，功能是单点动作
- Agent 执行**任务**，任务是目标导向的链路

Agent 要读取上下文、拆解步骤、调用工具、处理中间结果、检查异常、交付结果。**这条线一旦跨过去，软件设计的基本方法就要重写。**

---

## 四、三点跨越的连锁反应

### 连锁一：输入变模糊 → 上下文成为基础设施

一句「帮我排一下这篇文章」，没有上下文就只能泛泛处理。

如果系统知道你的公众号风格、排版方法论、历史文章、读者偏好，做出来的结果完全不一样。

> **Memory 从一个附属功能升级成了基础设施。**

### 连锁二：输出变开放 → Eval 极其重要

固定输出时代测试很简单：看按钮能不能点、字段对不对。

开放输出时代，问题变成：文章好不好？方案能不能打动人？判断是否可靠？

> **Eval 成为 Agent 软件的新测试系统，也是产品进化的方向盘。**

### 连锁三：执行变多步 → 设计对象从按钮路径变成任务链路

一个工具只需要做好一件事。一个 Agent 要判断：

- 什么时候做？
- 先后顺序是什么？
- 异常怎么处理？
- 什么地方需要人确认？

---

## 五、长尾，更多的长尾

当 AI 以极低成本进行开发的时候，更多的长尾需求被满足了。

过去，软件只服务高频、标准化、大规模的需求。太小的需求往往受制于 ROI 而不会被满足——除非是老板自己的。

比如：运营希望每星期自动盘点曝光数据。过去只能人工手动处理；现在，AI 让这些可以**即兴软件化**。

### 软件开始学习用户的工作方法

传统软件要求用户适应系统结构。AI 软件可以学习用户的表达、偏好、文件、风格、流程和判断方式。

> 软件个性化从「给你推荐什么」升级成了「学会你如何做事」。

### 软件开始临时生成自己的子软件

- 用户要看交互结果 → Agent 生成一个临时网页
- 用户要批量处理图片 → Agent 生成一个批处理工具

这些子软件存在的时间比一次会议还短，**用完即弃**。

---

## 六、开发者的价值在哪里？

回到开头的问题：当代码变成一次性的，开发者的价值在哪里？

过去开发者卖代码，卖的是**把需求翻译成指令的能力**。现在这个翻译过程 AI 可以做了。

当软件本身变成一次性的，很多卖软件的方法就不成立了。

或许更持久的生意，是让软件**更深入到客户的关系**：

- 更懂你的思考方式
- 更懂你的生活偏好
- 更懂你的工作流程

> 开发者从「翻译者」变成「编排者」——编排意图、链路、上下文、评估和权限。
